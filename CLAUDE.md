## 0. Project identity

**Name:** FaceRoots
**Tagline:** *Discover where your face comes from.*
**One-liner:** A mobile-first PWA where a user uploads portraits of themselves and their family, and the app produces a playful, narrated "Family DNA Map" showing which facial feature resembles which relative and by how much.

**This is a fun consumer app, not a forensic or genetic tool.** Every piece of copy, every LLM prompt, every share card must reinforce that framing. See §14 for the hard rules.

**Owner:** Swastik (solo build; Lovable scaffold → Claude Code for everything else via GitHub sync).

---

## 1. Non-negotiables — read before writing any code

1. **One Lovable prompt, ever.** Lovable has already generated the scaffold. Do not instruct the user to run any further Lovable prompts. All subsequent work ships via Claude Code → GitHub → Lovable sync.
2. **Privacy first.** Face data is biometric data under India's DPDP Act and GDPR. No feature ships without the consent flow wired. No raw image leaves the device until consent is recorded.
3. **pgvector, not Pinecone.** Stay inside Supabase. Do not introduce a second vector store.
4. **Edge Functions, not a FastAPI service.** All backend logic runs in Supabase Edge Functions (Deno). Heavy ML runs via hosted APIs (Replicate, fal.ai, Google AI Studio), not self-hosted.
5. **Never claim genetic relationship.** The app measures *visual resemblance*, not genetics. This phrasing is enforced in LLM system prompts, UI copy, share cards, and legal text.
6. **No ethnicity, attractiveness, health, or emotion commentary** from the LLM narration layer. Ever. Hard guardrail in every prompt.
7. **RLS on every table.** No exceptions. Every row scoped to `auth.uid()`.
8. **Model version stamping.** Every analysis row records the exact model versions used so results stay explainable across upgrades.
9. **18+ self-attestation.** v1 ships adults-only. Minors flow is a later decision with legal review.
10. **Kill switches.** Every paid API call sits behind a feature flag and a per-user rate limit.

## 4. Data model

All tables live in schema `public`. Every table has RLS enabled and a policy that scopes rows to `auth.uid()` via `owner_user_id` or via a join through `persons.owner_user_id`.

### 4.1 Core tables

```sql
-- Profiles (extends auth.users)
create table profiles (
  id uuid primary key references auth.users(id) on delete cascade,
  display_name text,
  avatar_url text,
  locale text default 'en',
  plan text default 'free' check (plan in ('free', 'pro')),
  age_attested_18_plus boolean default false,
  age_attested_at timestamptz,
  created_at timestamptz default now()
);

-- Family tree: the user IS a person; family members are persons too
create table persons (
  id uuid primary key default gen_random_uuid(),
  owner_user_id uuid not null references auth.users(id) on delete cascade,
  display_name text not null,
  relationship_tag text not null,
  -- 'self'|'mother'|'father'|'maternal_grandma'|'maternal_grandpa'
  -- |'paternal_grandma'|'paternal_grandpa'|'sibling'|'uncle'|'aunt'|'child'|'other'
  generation int not null default 0,
  -- 0=self, +1=parent, +2=grandparent, -1=child
  is_self boolean not null default false,
  birth_year_approx int,
  created_at timestamptz default now(),
  constraint one_self_per_user unique (owner_user_id, is_self)
    deferrable initially deferred
);
create index on persons (owner_user_id);

-- Raw face images (bucket: face-images, encrypted, TTL lifecycle)
create table face_images (
  id uuid primary key default gen_random_uuid(),
  person_id uuid not null references persons(id) on delete cascade,
  storage_path text not null,
  angle text not null check (angle in ('front', 'left', 'right', 'unknown')),
  capture_method text not null check (capture_method in ('guided_capture', 'upload_cropped')),
  width int, height int,
  blur_score float,       -- Laplacian variance, higher = sharper
  face_confidence float,  -- MediaPipe confidence 0..1
  nsfw_score float,       -- 0..1
  expires_at timestamptz, -- null = keep, else soft-delete target
  created_at timestamptz default now()
);
create index on face_images (person_id);
create index on face_images (expires_at) where expires_at is not null;

-- MediaPipe 468 landmarks + pose, kept for re-cropping with new models
create table face_landmarks (
  id uuid primary key default gen_random_uuid(),
  face_image_id uuid not null references face_images(id) on delete cascade,
  landmarks_json jsonb not null,
  pose_yaw float, pose_pitch float, pose_roll float,
  created_at timestamptz default now()
);

-- Holistic face embedding (one per face_image, from InsightFace)
create table face_embeddings (
  id uuid primary key default gen_random_uuid(),
  person_id uuid not null references persons(id) on delete cascade,
  face_image_id uuid not null references face_images(id) on delete cascade,
  embedding vector(512) not null,   -- ArcFace buffalo_l
  quality_score float,              -- InsightFace face quality
  model_version text not null,      -- e.g. 'buffalo_l@2024-02'
  created_at timestamptz default now()
);
create index on face_embeddings using hnsw (embedding vector_cosine_ops);
create index on face_embeddings (person_id);

-- Per-feature crops → DINOv2 embeddings (the matchable data)
create table feature_embeddings (
  id uuid primary key default gen_random_uuid(),
  person_id uuid not null references persons(id) on delete cascade,
  face_image_id uuid not null references face_images(id) on delete cascade,
  feature_type text not null,
  -- 'eyes_left'|'eyes_right'|'nose'|'mouth'|'jawline'|'forehead'
  -- |'eyebrows_left'|'eyebrows_right'|'ear_left'|'ear_right'
  -- |'hairline'|'face_shape'
  crop_storage_path text,
  embedding vector(384) not null,   -- DINOv2 ViT-S/14
  quality_score float,
  model_version text not null,      -- e.g. 'dinov2-vits14@2024-01'
  created_at timestamptz default now()
);
create index on feature_embeddings using hnsw (embedding vector_cosine_ops);
create index on feature_embeddings (person_id, feature_type);

-- Analyses (a "run" that produces a DNA map)
create table analyses (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references auth.users(id) on delete cascade,
  self_person_id uuid not null references persons(id),
  status text not null default 'pending'
    check (status in ('pending','embedding','matching','narrating','rendering','done','failed')),
  error_message text,
  model_versions jsonb,  -- {face: '...', features: '...', llm: '...'}
  started_at timestamptz default now(),
  completed_at timestamptz
);
create index on analyses (user_id, started_at desc);

-- Match results: per feature, winner + runners-up
create table feature_matches (
  id uuid primary key default gen_random_uuid(),
  analysis_id uuid not null references analyses(id) on delete cascade,
  feature_type text not null,
  winner_person_id uuid references persons(id),
  winner_similarity float,           -- 0..1
  winner_confidence float,           -- computed across 3 angles when available
  runners_up jsonb,                  -- [{person_id, similarity}, ...]
  llm_verdict text,                  -- playful one-liner
  created_at timestamptz default now()
);
create index on feature_matches (analysis_id);

-- Consent audit trail (DPDP/GDPR)
create table consent_events (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references auth.users(id) on delete cascade,
  event_type text not null,          -- 'granted'|'revoked'|'updated'
  scopes jsonb not null,             -- {embeddings:true, raw_images:false, sharing:true}
  policy_version text not null,
  user_agent text,
  ip_hash text,                      -- sha256(ip + daily_salt), never raw IP
  created_at timestamptz default now()
);

-- Rate limiting
create table rate_limit_events (
  user_id uuid not null references auth.users(id) on delete cascade,
  action text not null,              -- 'run_analysis'|'embed_features'|...
  created_at timestamptz default now()
);
create index on rate_limit_events (user_id, action, created_at desc);
```

### 4.2 RLS policy pattern

Every table follows this shape:

```sql
alter table <t> enable row level security;

create policy "<t>_owner_select" on <t>
  for select using (owner_user_id = auth.uid());
create policy "<t>_owner_insert" on <t>
  for insert with check (owner_user_id = auth.uid());
create policy "<t>_owner_update" on <t>
  for update using (owner_user_id = auth.uid());
create policy "<t>_owner_delete" on <t>
  for delete using (owner_user_id = auth.uid());
```

For tables without `owner_user_id` (face_images, face_embeddings, feature_embeddings, face_landmarks), policies join through `persons.owner_user_id`.

### 4.3 Storage buckets

| Bucket | Public? | Contents | TTL |
|---|---|---|---|
| `face-images-raw` | Private | Full portraits | Default 7 days, user-configurable 1/7/keep |
| `feature-crops` | Private | Per-feature PNGs | Same lifecycle as face-images-raw |
| `legacy-cards` | Private | Rendered share PNGs | 30 days |

All access via signed URLs with ≤15 min expiry. Bucket policies mirror table RLS.

## 5. Edge Functions

All Deno, located in `supabase/functions/<name>/index.ts`. Shared code in `supabase/functions/_shared/`. Every function: validates JWT, checks rate limits, logs to Sentry, returns typed JSON.

### 5.1 `validate-face`
**Input:** `{ face_image_id }` (image already uploaded by client)
**Output:** `{ valid: boolean, reason?: string, face_confidence: number, nsfw_score: number }`
**Logic:** Download image → run server-side face detection (Replicate `retinaface` or similar, as a trust check on client claim) → run NSFW classifier → write scores to `face_images` → return verdict. Reject if `face_confidence < 0.85`, `nsfw_score > 0.3`, or no face found.

### 5.2 `embed-face`
**Input:** `{ face_image_id }`
**Output:** `{ embedding_id }`
**Logic:** Fetch image → call Replicate InsightFace (`buffalo_l`) → store 512-dim vector with `model_version`. Idempotent: returns existing row if one exists for this image.

### 5.3 `embed-features`
**Input:** `{ face_image_id, crops: [{feature_type, storage_path}, ...] }`
**Output:** `{ count }`
**Logic:** For each crop, call DINOv2 endpoint, store 384-dim vector in `feature_embeddings`. Fans out with `Promise.all` but caps concurrency at 4. Retries with exponential backoff (max 3).

### 5.4 `run-analysis` (orchestrator)
**Input:** `{ self_person_id }`
**Output:** `{ analysis_id }` — client subscribes via Supabase Realtime for status updates
**Logic:**
1. Create `analyses` row with status=`pending`
2. Verify self has ≥1 front angle image with embeddings
3. Verify ≥1 family member has embeddings
4. Advance status → `matching` → call match logic (§5.5)
5. Advance status → `narrating` → call narration (§5.6)
6. Advance status → `rendering` → call legacy card render (§5.7)
7. Mark `done` with `completed_at` and `model_versions`

Wraps everything in a try/catch that sets status=`failed` with `error_message` on any error.

### 5.5 `match-features`
**Input:** `{ analysis_id }`
**Output:** `{ matches_written: number }`
**Logic:** For each feature_type the self has across all 3 angles:
```sql
select person_id, avg(1 - (embedding <=> $user_avg)) as similarity
from feature_embeddings
where person_id in (select id from persons where owner_user_id = $user and not is_self)
  and feature_type = $feature
group by person_id
order by similarity desc
limit 5;
```
Uses the **mean embedding** across the self's 3 angles for robustness. Writes top-1 as winner + up to 4 runners-up. Computes `winner_confidence` from the std-dev across angles (tight cluster = high confidence).

### 5.6 `narrate-matches`
**Input:** `{ analysis_id }`
**Output:** `{ narrated: number }`
**Logic:** For each `feature_match`, send two cropped images (user's + winner's) to Gemini 2.5 Flash with the locked system prompt (§9.2). Cache verdicts by `(user_crop_hash, winner_crop_hash)` to avoid re-billing. Write `llm_verdict` back to `feature_matches`.

### 5.7 `render-legacy-card`
**Input:** `{ analysis_id }`
**Output:** `{ signed_url }`
**Logic:** Compose 1080×1920 PNG via Satori (React-to-SVG) → resvg-js (SVG→PNG). Upload to `legacy-cards` bucket, return 15-min signed URL.

### 5.8 `delete-my-data`
**Input:** (authed, no body)
**Output:** `{ deleted: {tables: count, storage: count} }`
**Logic:** Hard-delete from all tables (cascades handle most), purge all storage objects for user, write one final `consent_events` row with `event_type='revoked'` then delete the profile. Returns receipt.

### 6.1 MediaPipe integration
- Load `FaceLandmarker` once at app boot, cache in a Zustand store.
- Model: `face_landmarker.task` from MediaPipe CDN.
- Run mode: `VIDEO` during capture, `IMAGE` during family upload.
- Output used: 478 3D landmarks, blendshapes, facial transformation matrix.

Cropping: compute bounding box from landmark indices, pad 15%, resize to 224×224 square, export as PNG blob.

### 6.3 3-angle capture flow
Single screen, three sequential steps, one camera stream. State machine:

```
idle → detecting_front → captured_front →
      detecting_left  → captured_left  →
      detecting_right → captured_right → done
```

Advance rules:
- **Front:** `|yaw| < 5° && |pitch| < 5°` stable for 1.5s
- **Left:** `yaw ∈ [+45°, +75°]` stable for 1.5s
- **Right:** `yaw ∈ [-75°, -45°]` stable for 1.5s
- Face must fill ≥15% of frame
- Blur score (Laplacian variance) ≥ 100

On capture: freeze frame, haptic buzz (`navigator.vibrate(30)`), shutter animation, upload in background while user proceeds to next angle.

Fallback path: if camera fails or user retries 3×, switch to "upload 3 photos" with the same overlay on static images.

### 6.4 Family member upload flow
1. File picker (accept `image/*`)
2. Run FaceLandmarker in IMAGE mode
3. If 0 faces → friendly error, show good/bad examples
4. If >1 face → show "tap the face you want" UI
5. Auto-crop to face bbox + 20% padding
6. Confirm screen: crop preview, name input, relationship dropdown
7. Upload → `validate-face` Edge Function → `embed-face` + `embed-features`

## 9. LLM prompts (locked)

### 9.1 System prompt — narration (Gemini 2.5 Flash)

```
You are writing playful, warm, one-line captions for a fun consumer app
that compares facial features between family members.

Rules you MUST follow:
- Write exactly one sentence, max 15 words.
- Be warm, light, a little witty. Never sarcastic or mean.
- Refer to the feature by name (eyes, nose, mouth, jawline, etc).
- You may use at most one emoji. None is also fine.
- NEVER comment on: attractiveness, beauty, ugliness, health, weight,
  age, emotion, mood, ethnicity, race, skin tone, or perceived gender.
- NEVER claim genetic, biological, or hereditary relationship. This is
  a visual resemblance game, not a DNA test.
- If the two crops look clearly different, say so honestly but kindly
  ("not quite a match on this one").
- Output ONLY the sentence. No preamble, no quotes, no markdown.
```

### 9.2 User prompt template

```
Feature: {feature_type}
Similarity score: {similarity_pct}%
Relationship: {relationship_tag}

[image 1: user's {feature_type} crop]
[image 2: {relationship_tag}'s {feature_type} crop]

Write the caption.
```

### 9.3 Guardrail post-filter

After the LLM responds, run a regex deny-list check for: `beautiful|ugly|pretty|handsome|attractive|dna|gene|genetic|hereditary|blood|ethnic|race|old|young|sick|healthy|fat|thin`. If any match, discard and fall back to a template like `"Your {feature} has a {similarity_pct}% echo of your {relationship_tag}."`

## 10. Validation pipeline (gibberish defense)

Layered, cheap-first:

1. **Client pre-upload:** face confidence ≥ 0.85, face/frame ratio ≥ 15%, blur score ≥ 100, single face. Free, instant.
2. **Server post-upload (`validate-face`):** independent face detection as trust check, NSFW classifier, EXIF sanity, file size ≤ 10MB, dims ≥ 400×400.
3. **Pose validation (self captures only):** the 3 uploaded angles must have pose_yaw values spanning ≥ 90° total. Otherwise reject with "please retry side profiles."
4. **Rate limiting:** 3 analyses/day free, 30 uploads/day, 100 validation calls/day. All keyed by user_id in `rate_limit_events`.

## 13. Coding conventions

- **TypeScript strict mode on.** No `any`. Use `zod` for all external boundaries.
- **One default export per component file**, named matching the file.
- **No default exports in utility files** — named exports only.
- **Server-first data:** use TanStack Query, no client-side caching of Supabase rows in Zustand. Zustand is for UI state only (modal open, capture step, etc).
- **Tailwind only.** No CSS modules, no styled-components. Use `cn()` helper from shadcn.
- **Animations via Framer Motion.** No CSS keyframes unless absolutely necessary.
- **Error handling:** every Edge Function wraps its handler in `withSentry(withRateLimit(handler))`. Every fetch in the frontend uses TanStack Query error boundaries.
- **Testing:** Vitest for pure functions (cropper, pose math, guardrail filter). Playwright for critical flows (auth → consent → capture → analysis → result).
- **Commits:** Conventional Commits. `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`.
- **Branches:** `main` is always deployable. Feature branches `phase-N/short-name`.

## 14. Tone & safety rules (hard)

These show up everywhere: UI copy, LLM prompts, share cards, error messages.

1. **Language:** "resemblance," "looks like," "echoes," "reminds us of." Never: "inherited," "genetic," "DNA," "bloodline."
2. **Legal disclaimer** on every results page, every share card, the consent screen, and the app footer: *"FaceRoots is a fun visual resemblance tool. It is not a genetic, paternity, or ancestry test and must not be used for any legal, medical, or identity purpose."*
3. **No attractiveness commentary.** Not in prompts, not in copy, not in achievements.
4. **No ethnicity or race commentary.** Not even positively framed.
5. **No minors.** v1 requires 18+ self-attestation at signup. Any flow for child photos is out of scope until legal review.
6. **Feedback loop:** every result card has a "This feels off" button that writes to a moderation queue for review.

## 15. Known risks and mitigations

Document live in `RISKS.md`. Initial entries:

| Risk | Likelihood | Mitigation |
|---|---|---|
| Face models have accuracy disparities across skin tones | High | Phase 6 internal audit; per-feature confidence surfacing; "this feels off" feedback |
| User uploads someone else's photo (non-consensual) | Medium | ToS warning at upload, no face-search across users, per-user RLS isolation |
| LLM outputs something mean or biased | Medium | Locked system prompt + regex post-filter + template fallback |
| Bad actors flood API via automation | Medium | Supabase Auth required, per-user rate limits, Cloudflare Turnstile on auth |
| Raw images leak via signed URL | Low | ≤15 min expiry, HTTPS only, TTL lifecycle, default 7-day purge |
| Model upgrade breaks old results | Low | `model_versions` stamped on every analysis; old results remain viewable |
| DPDP/GDPR compliance gap | Medium | Consent events logged, deletion endpoint, legal review before launch |

## 16. What Claude Code should never do

- Run any Lovable prompt (scaffold is already done).
- Introduce FastAPI, Pinecone, Qdrant, AWS Rekognition, or Azure Face.
- Store raw credit card, government ID, or medical data.
- Ship a feature without updating RLS policies.
- Ship copy or LLM output that references genetics, ethnicity, attractiveness, or health.
- Add dependencies without checking bundle size impact (mobile-first — every KB matters).
- Break the one-file-per-component convention without discussion.
- Deploy to production without a migration plan for the schema changes in that PR.
